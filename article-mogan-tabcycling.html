Mogan 数学模式 TabCycling 细节复盘

2025-10-01 郑梓辉

项目背景

Mogan 是一个支持科学文档编辑的项目，其中数学模式是核心功能之一。 TabCycling 是一个重要的用户交互特性，允许用户通过 Tab 键在不同的数学元素或占位符之间快速导航。

核心需求分析

TabCycling 的实现需要解决以下问题：

  • 识别当前可焦点的数学元素
  • 维护正确的焦点顺序（从左到右、从上到下）
  • 支持 Shift+Tab 反向循环
  • 与输入法模式的协调

实现细节

1. 焦点图的构建

在数学表达式树中，我们需要维护一个焦点图，记录所有可获得焦点的节点及其顺序。

2. 插入模式中的图片插入

在插入模式下，用户可以通过特定快捷键插入图片。核心挑战是正确管理：

  • 图片的大小和位置约束
  • 与周围数学元素的对齐
  • 插入后的焦点位置

3. Backspace 和 Delete 的处理

删除操作需要考虑多种情况：

  • 删除单个字符：简单情况，直接删除
  • 删除选中内容：替换为占位符或空值
  • 删除整个表达式：需要回溯到上一级编辑状态
  • 边界情况：在表达式开头或结尾删除时的焦点管理
// 伪代码示例
function handleDelete(cursor, node) {
    if (cursor.atStart() && node.parent) {
        // 删除当前节点，焦点回到父节点
        node.parent.removeChild(node);
        cursor.moveToParent();
    } else {
        // 删除光标处的内容
        node.deleteAt(cursor.position);
    }
}

踩坑与教训

陷阱 1：焦点顺序不一致

初期的实现中，焦点顺序与用户的直观期望不符，尤其在嵌套表达式中。 解决方案是采用深度优先遍历，并加入位置坐标作为辅助排序键。

陷阱 2：输入法与焦点的冲突

在输入法激活时，Tab 键可能被拦截，导致 TabCycling 失效。 我们通过显式处理输入法事件和焦点事件的优先级来解决这个问题。

测试策略

  • 单元测试：验证焦点图构建的正确性
  • 集成测试：验证 Tab 导航的完整流程
  • 手工测试：验证各种嵌套和边界情况

总结

完成 TabCycling 的实现让我更深入地理解了编辑器的事件系统和焦点管理机制。 同时，这个工作也为后续的其他交互特性提供了基础。
